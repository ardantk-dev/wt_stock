# 📈 국내외 주식관리 텔레그램 에이전트

본 에이전트는 사용자의 국내외 보유 주식을 관리하고, 텔레그램을 통해 매일 아침 시장 분석 및 저녁 종가 브리핑을 자동으로 발송합니다. 또한 당일 매매 기록을 텔레그램 채팅으로 간단하게 입력받아 포트폴리오를 업데이트합니다.

---

## 🛠️ 준비 및 설정 방법

### 1단계: 텔레그램 봇 생성 및 토큰 발급
1. 텔레그램 앱에서 **@BotFather**를 검색하여 대화방을 엽니다.
2. `/newbot` 명령어를 입력합니다.
3. 봇의 이름(Name)과 사용자 이름(Username, `_bot`으로 끝나야 함)을 설정합니다.
4. 생성이 완료되면 `HTTP API Token`이 발급됩니다. (예: `123456789:ABCdefGhIJKlmNoPQRsTUVwxyZ`)

### 2단계: 에이전트 설정 파일 업데이트
1. 본 프로젝트 폴더에 생성된 `config.json` 파일을 엽니다.
2. `telegram_token` 필드에 발급받은 토큰 값을 복사하여 입력하고 저장합니다.
```json
{
  "telegram_token": "발급받은_API_토큰_값",
  "telegram_chat_id": "",
  "morning_time": "07:00",
  "evening_time": "20:30"
}
```

### 3단계: 에이전트 실행 및 Chat ID 자동 등록
1. 터미널(Command Prompt 또는 PowerShell)에서 아래 명령어를 실행하여 에이전트를 가동합니다.
   ```bash
   python run.py
   ```
2. 에이전트가 가동되면 텔레그램에서 자신이 생성한 봇(Username 검색) 대화방으로 들어가 **/start** 명령어를 보냅니다.
3. 에이전트가 사용자의 메시지를 감지하고 **Chat ID를 자동으로 인식하여 `config.json`에 저장**합니다. 이후부터 스케줄러 브리핑이 지정된 시간에 자동으로 발송됩니다.

---

## 📊 주요 기능 및 사용법

### 1. 텔레그램 명령어 및 버튼 메뉴
봇 대화방 하단에 제공되는 메뉴 버튼을 클릭하거나 명령어를 직접 입력할 수 있습니다.
*   **📊 포트폴리오 조회** (`/portfolio`): 보유 중인 국내외 주식의 평가 금액, 실시간 등락, 총 수익률 및 평가 손익 현황을 조회합니다.
*   **🌅 아침 브리핑** (`/morning`): 오전 7시에 발송되는 야간 선물, 미 증시 마감 지표, 국채 금리, 환율 및 보유 종목 뉴스 브리핑을 수동으로 미리 받아봅니다.
*   **🌆 저녁 브리핑** (`/evening`): 오후 8시 30분에 발송되는 국내 증시 마감 요약 및 보유 자산 수익 현황 브리핑을 수동으로 받아봅니다.
*   **`/add [종목명/티커] [수량] [평단가]`**: 보유 종목을 포트폴리오에 수동으로 직접 등록합니다. (예: `/add 삼성전자 10 75000`, `/add AAPL 5 180`)
*   **`/remove [종목명/티커]`**: 포트폴리오에서 특정 종목을 제거합니다.

## ✍️ 간편 매매 기록 등록 가이드
증권사(키움증권, 토스증권 등)는 보안 정책 및 기술적 제약으로 인해 백그라운드 스크립트에서 실시간 자동 연동이 불가능합니다. 
대신, 텔레그램 창에 **단 한 줄의 메시지를 치는 것만으로 평단가와 보유량이 자동 계산**되도록 구현하였습니다.

매매를 진행한 날 저녁, 봇 대화창에 아래 형식으로 채팅을 보내기만 하면 됩니다.

### 입력 양식
> **`[종목명 또는 티커] [매수/매도] [수량] [체결가]`**

*   **국내 주식 매수 예시**: `삼성전자 매수 10 74500`
    *   (기존 보유량에 수량이 더해지며 평단가가 가중 평균값으로 자동 갱신됩니다.)
*   **해외 주식 매수 예시**: `AAPL 매수 5 185.2`
*   **국내 주식 매도 예시**: `sk하이닉스 매도 3 182000`
    *   (보유량에서 3주가 차감됩니다. 만약 보유량 전체를 매도하면 포트폴리오에서 자동 삭제됩니다.)
*   **해외 주식 매도 예시**: `TSLA 매도 10 178.5`

## 🕒 스케줄러 자동 실행 (백그라운드 작동)
에이전트가 24시간 동안 작동하며 오전 7시와 오후 8시 30분에 브리핑을 안정적으로 보내기 위해서, 로컬 PC나 서버에서 `python run.py`가 상시 켜져 있어야 합니다.

### Windows에서 백그라운드로 실행하기 (선택 사항)
명령 프롬프트(CMD) 창을 띄워두지 않고 백그라운드 프로세스로 띄우고 싶다면 아래 명령어로 실행할 수 있습니다.
```powershell
# 백그라운드 프로세스로 실행
Start-Process python -ArgumentList "run.py" -WindowStyle Hidden
```
프로세스를 종료하고 싶을 때는 작업 관리자(Task Manager)에서 `python` 프로세스를 종료하시면 됩니다.
